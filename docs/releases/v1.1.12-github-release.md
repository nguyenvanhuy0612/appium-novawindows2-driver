# v1.1.12

Hot-fix for [1.1.11](https://github.com/nguyenvanhuy0612/appium-novawindows2-driver/releases/tag/v1.1.11), which hung at session start.

> npm: https://www.npmjs.com/package/appium-novawindows2-driver/v/1.1.12

## Why this exists

**1.1.11 hung at session start for the full 60 s `powerShellCommandTimeout` window**, then surfaced:

```
PowerShell init stage 'using-namespace' failed: PowerShell command timed out after 60000ms
```

Reproducible against the target VM with a stripped-down repro: variant `bare using namespace` works; variant `& { using namespace System.Windows.Automation }` exits with zero stdout/stderr тАФ the parser silently never completes.

## What 1.1.11 did that broke

1.1.11 wrapped every PS command in `& { ... }` to prevent multi-line constructs from absorbing the marker lines. Two breakages followed:

1. **`using namespace` inside `& { ... }` hangs PowerShell's stdin parser.** The init script never completes; every subsequent command hits the timeout.
2. Variables defined inside `& { }` (`$cacheRequest`, `$elementTable`, `$rootElement`, тАж) are local to that subshell and vanish on return. Every command depended on those persisting.

## Fix

- **Drop the `& { ... }` wrap.** User commands are sent raw between the `$LASTEXITCODE = 0` reset and the marker lines.
- **Consolidate init back to one combined script** sent as a single `sendPowerShellCommand` call, so `using namespace` at the top applies to the whole script and assemblies + variables persist correctly.
- Drop the per-init-stage timing logs (the named-stages were the observability that drove the original split тАФ not worth the breakage).

**Trade-off accepted**: a malformed user command (open pipe, unclosed brace) will now absorb the marker lines and surface as a 60 s timeout rather than silent corruption. That's the caller's bug, made visible. Better to fail loudly on bad input than break legitimate scripts.

## Everything that 1.1.11 added is still here

The 1.1.11 release notes catalogue what landed тАФ UUID markers, dual stdout/stderr completion, `treatStderrAsError`, `$LASTEXITCODE` detection, per-command buffer isolation, `powerShellTerminating` flag. **All of it is still in 1.1.12** тАФ only the `& { ... }` wrap was rolled back.

## Lesson for the release process

The `using namespace`-inside-`& { }` parser hang is the kind of bug that **unit tests using a fake `ChildProcess` cannot catch**. The next stable release should run the E2E suite against a real PowerShell in CI on a Windows runner.
