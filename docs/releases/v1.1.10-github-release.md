# v1.1.10

Hotfix release — fixes a critical install bug in 1.1.9.

> npm: https://www.npmjs.com/package/appium-novawindows2-driver/v/1.1.10
> Full notes: [`CHANGELOG.md`](https://github.com/nguyenvanhuy0612/appium-novawindows2-driver/blob/v1.1.10/CHANGELOG.md#1110-2026-05-08)

## Why this exists

**1.1.9 published to npm fails to load on a fresh `appium driver install`**:

```
[Appium] Could not load driver 'novawindows2', so it will not be available.
[Appium] Error: Cannot find module 'asyncbox'
Require stack:
- C:\Users\admin\.appium\node_modules\appium-novawindows2-driver\build\lib\commands\screen-recorder.js
```

The driver worked locally (the dev `node_modules` tree happens to satisfy these via Appium's own deps) but installs from npm only get declared deps.

## Fix

`asyncbox` and `teen_process` were imported eagerly at the top of `lib/commands/screen-recorder.ts` but never declared in `package.json`. Compounded by the project's ARM64 design constraint: there's no prebuilt FFmpeg binary for Win-ARM64, so the **whole recording stack should be fully optional**.

Resolved with the **lazy + `optionalDependencies`** pattern matching `ffmpeg-static`:

- `asyncbox` and `teen_process` moved to `optionalDependencies` alongside `ffmpeg-static`
- `screen-recorder.ts` switched to lazy-`require()` via a new `loadRecordingDeps()` helper called inside `start()` rather than at module load
- If any of the three packages are missing, only the recording functions fail (with an actionable error pointing at the missing dep). The rest of the driver loads cleanly

## Behaviour matrix

| Scenario | Result |
| :--- | :--- |
| Win-x64 with full recording stack | ✅ recording works |
| Win-x64, npm hiccup → `asyncbox` missing | ✅ driver loads; `windows: startRecordingScreen` throws clean error |
| Win-ARM64, recording deps intentionally skipped | ✅ driver loads; recording calls throw clean error |
| Win-ARM64, deps installed but no ARM FFmpeg binary | ✅ driver loads; recording call throws "ffmpeg-static binary missing or not supported on this architecture" |
