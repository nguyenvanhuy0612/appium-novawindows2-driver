## Highlights

### 🚀 PowerShell auto-restart

`sendPowerShellCommand` now detects a dead persistent PS session (process exited or stdin not writable) and transparently re-runs `startPowerShellSession()` before queueing the next command. Concurrent callers share a single `powerShellRestartPromise` so a burst of queued commands triggers exactly one restart. Replaces the previous failure mode where every command after a PS crash returned `"PowerShell session is not available or closed"` until `deleteSession`.

Stale element IDs surface as `NoSuchElementError` after restart (correct behaviour). `prerun` is **not** re-run.

### 🔒 Security: `pushFile` payload validation

`base64Data` is now validated against `^[A-Za-z0-9+/=\s]*$` before interpolation into PowerShell. Closes a PS-injection vector where a crafted "base64" payload could break out of the single-quoted string and execute arbitrary commands in the persistent session.

### 🆕 W3C standard commands (from upstream sync)

Added at the driver level: `title`, `maximizeWindow`, `minimizeWindow`, `back`, `forward`, `setWindowRect`, `closeApp`, `launchApp`. `setWindowRect` is backed by new `TransformPattern` Move/Resize templates. WebdriverIO / Appium-Python-Client now hit these via the standard wire protocol — no `windows:` prefix needed.

## Bug fixes

| Area | Fix |
|---|---|
| **EPIPE crash** | Added `stdin.on('error')` and `process.on('exit')` handlers — a dying PS process no longer crashes Node with an unhandled `EPIPE`. |
| **Rect dimensions** | `Math.min(p2[0]-p1[0])` (single-arg, returns input unchanged) → `Math.abs(...)`. `Rect(point, point)` selectors now build positive-dimensioned rects regardless of point order. |
| **`setWindow` foreground** | Was calling `trySetForegroundWindow(NaN)` on name-based lookup. Now reads the resolved element's `NATIVE_WINDOW_HANDLE` and passes that. |
| **Pattern handlers** | 8 handlers (`patternIsMultiple`, `patternGet/AllSelectedItems`, `patternAdd/RemoveFromSelection`, `patternSet/GetValue`, `patternScrollIntoView`) bypassed `resolvePatternElement`; stale IDs surfaced as raw PS errors instead of W3C `NoSuchElementError`. All routed through the resolver now. |
| **UWP path heuristic** | `path.includes('!') && path.includes('_')` was too loose. Replaced with `^[\w.-]+_[A-Za-z0-9]+![\w.-]+$` matching real AUMID shape. |
| **`NotCondition` error** | Said "AndCondition expects Conditions as args" inside `NotCondition`. Fixed. |
| **`$()` validation** | `&&` should be `||`. Constructor masked it. |
| **`handleMouseMoveAction`** | Re-fetches rect after `scrollIntoView` instead of using stale off-screen coords. |
| **`setValue` modifiers** | Right-hand modifier was released wrongly when paired with left-hand of same type. Now release-on-throw is wrapped in try/finally. |
| **Booleans** | `elementSelected` strict-case `=== 'True'`/`=== 'On'` aligned to case-insensitive across `elementDisplayed` / `elementEnabled` / `patternIsMultiple`. |
| **`patternSetValue` errors** | Both ValuePattern and RangeValuePattern errors now composed when both fail. |
| **`pushCacheRequest`** | Named `treeScope`/`automationElementMode` validation never ran (regex had no named captures, `&&` should have been `||`). Both fixed. `treeFilter` parse errors now `InvalidArgumentError` instead of `InvalidSelectorError`. |

## Infrastructure

- **`scripts/local/build_deploy_restart.ps1` rewritten**: extracted `Invoke-RemotePs` / `Invoke-RemoteScp` helpers; added `-SkipBuild`, `-SkipInstall`, `-NoRestart`, `-RestartOnly` flags; auto-counted step numbering; per-step `$LASTEXITCODE` validation; remote scripts use `Write-Output` (no CLIXML noise / duplicate-line output); new `npm install --omit=dev` step on remote (fixes first-deploy missing-`@appium/base-driver` failure).
- **`scripts/local/copy_log.ps1` rewritten**: parameterized `-RemoteHost` / `-RemoteUser`; added `-Tail N` and `-Follow` modes.
- **`scripts/local/Restart_Appium_Remotely.ps1` deleted** (superseded by `-RestartOnly`).
