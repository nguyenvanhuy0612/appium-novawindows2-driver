# v1.1.13

Patch release: XPath `floor()` / `ceiling()` / `round()` argument coercion fix per W3C §4.4, plus an expanded E2E test suite and three design-review documents guiding the path to the next stable cut.

> npm: https://www.npmjs.com/package/appium-novawindows2-driver/v/1.1.13

## User-visible

### `fix(xpath)`: implicit `number()` coercion for `floor` / `ceiling` / `round`

XPath 1.0 §4.4 specifies that `floor(x)`, `ceiling(x)`, and `round(x)` take a **number**, with non-number arguments implicitly converted via `number()`. Previously the driver threw `InvalidArgumentError` for any argument with `typeof !== 'number'`:

```ts
// Before (1.1.12 and earlier):
//Element[floor(@SomeNumericProperty) > 0]   →   InvalidArgumentError
//Element[round(string-length(@Name) div 2)] →   InvalidArgumentError (sometimes — depends on PS return shape)

// After (1.1.13):
floor("3.7")    → 3       (string coerced via number())
ceiling("3.2")  → 4
round("3.5")    → 4
floor(true())   → 1       (boolean → 1/0)
floor("abc")    → NaN     (unparseable → NaN, then Math.floor(NaN) === NaN)
floor(<element>) → NaN    (element → empty string → NaN)
```

10-line fix at [`lib/xpath/functions.ts:144-160`](https://github.com/nguyenvanhuy0612/appium-novawindows2-driver/blob/v1.1.13/lib/xpath/functions.ts#L144-L160) + 9 new unit tests in [`tests/unit/xpath-comprehensive.spec.ts`](https://github.com/nguyenvanhuy0612/appium-novawindows2-driver/blob/v1.1.13/tests/unit/xpath-comprehensive.spec.ts).

This closes 3 of the 16 pre-existing E2E failures from the [1.1.10 smoke run](https://github.com/nguyenvanhuy0612/appium-novawindows2-driver/blob/v1.1.13/docs/development/deep-architecture-review.md#5-triage-of-the-16-pre-existing-e2e-failures); the remaining 13 are test-code drift against WDIO v9's API changes or Win11 Notepad's UI layout — fixable with a mechanical refresh of `tests/e2e/xpath.e2e.spec.ts`.

## Additive (no user-visible runtime change)

### 8 new E2E suites under `tests/e2e/stable/` (82 tests, ~95 s end-to-end)

| Suite | Tests | What it covers |
|---|---|---|
| `lifecycle.e2e.spec.ts` | 4 | Session create / delete / `shouldCloseApp` / NoSuchDriver-after-teardown |
| `error-surface.e2e.spec.ts` | 5 | NoSuchElement, InvalidSelector, StaleElement, runaway-PS timing |
| `scroll-focus.e2e.spec.ts` | 3 | `windows: setFocus` + `windows: scrollIntoView` no-hang guarantees |
| `stderr-and-native.e2e.spec.ts` | 5 | `treatStderrAsError` + `$LASTEXITCODE` native-exit detection |
| `stress.e2e.spec.ts` | 6 | 50 sequential / 20 parallel finds, queue ordering, 100 PS round-trips |
| `unicode.e2e.spec.ts` | 7 | PSString escaping, UTF-8 transport, `[char]0xNNNN` codepoint construction, clipboard |
| `desktop.e2e.spec.ts` | 23 | XPath axes / predicates / functions against real Windows shell elements |
| `component-coverage.e2e.spec.ts` | 29 | One smoke test per public command (breadth, not depth) |

The desktop and component-coverage suites use `app: 'Root'` (Windows desktop) with **locale-invariant `@AutomationId` selectors** captured from a real desktop snapshot — they don't depend on a specific app being installed or a specific UI state, and run ~10× faster than Notepad-anchored tests (no app launch).

Lessons learned during 9 rounds of triage are baked into code comments inline so future maintainers don't have to rediscover them — most notably:

- WDIO v9 changed the API shape for several Element methods (`getElementRect` / `getRect`, `getDeviceTime`, `launchApp`); use the `windows:` extension or the PS escape hatch as workarounds
- Pattern extension commands (`windows: invoke` / `setValue` / `getValue` / …) read `element[W3C_ELEMENT_KEY]`, not `element.elementId` — pass the WDIO Element object directly
- Win11 Notepad's Document control does NOT expose ValuePattern (set/get are silent no-ops); use `[char]0xNNNN` construction for unicode payloads sent via PS
- The bare `powershell` script name routes through `executePowerShellScript`; `windows: powershell` does NOT route there (it falls through to the EXTENSION_COMMANDS table which doesn't include it)

## Upgrade

```sh
appium driver uninstall novawindows2
appium driver install --source=npm appium-novawindows2-driver@latest
```

Safe to upgrade from 1.1.12 or 1.1.10. **Skip 1.1.11** (deprecated; hangs at session start).
