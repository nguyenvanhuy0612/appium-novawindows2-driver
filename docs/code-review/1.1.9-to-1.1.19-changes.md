## 1.1.9 — `a46c3ad` "PS auto-restart, code-review hardening, docs reorg"

Gộp 2 batch lớn. Đây là release thay đổi nhiều nhất kể từ 1.1.8.

### Batch A — Upstream sync (gốc tag `1.2.0` nội bộ, commit `99b016b`)

**Features mới (có thể là nguồn regression):**
- Thêm 8 W3C driver-level commands: `title`, `maximizeWindow`, `minimizeWindow`, `back`, `forward`, `setWindowRect`, `closeApp`, `launchApp`.
- `TransformPattern` support: thêm `MOVE_WINDOW` / `RESIZE_WINDOW` template + `buildMove` / `buildResizeCommand` (backing `setWindowRect`).
- Wire `windows: launchApp` / `windows: closeApp` vào `EXTENSION_COMMANDS`.
- Helper mới: `parseRectJson()`, `withModifierKeys()`, `ensureElementResolved()`.

**Bug fixes:**
- `handleMouseMoveAction`: re-fetch rect sau `scrollIntoView` (trước parse rect cũ).
- `setValue`: bấm L_SHIFT rồi R_SHIFT trước đây thả nhầm variant → leak key. Fix cho cả Ctrl/Alt/Meta.
- `setValue`: bọc modifier release trong try/finally.
- `elementDisplayed` / `elementSelected` / `elementEnabled` / `patternIsMultiple`: coerce boolean case-insensitive nhất quán.
- `patternSetValue`: compose error từ cả ValuePattern và RangeValuePattern.
- `pushCacheRequest`: fix `.groups?.[0]` trên regex unnamed group (luôn undefined) + đổi `&&` → `||` cho range check.

**Refactor:**
- Export `ModifierKeyName`, thêm `MODIFIER_KEY_MAP`, `CLICK_TYPE_BUTTON_MAP`.
- Extract `toggleModifier()` closure trong `setValue`.
- Rename `$cx` / `$cy` → `$centerX` / `$centerY` (chỉ trong `GET_ELEMENT_LEGACY_PROPERTY`).

### Batch B — Code-review hardening (cùng session)
- **PowerShell auto-restart**: `ensurePowerShellSession` tự động re-spawn PS session đã chết, concurrent callers dedupe qua `powerShellRestartPromise`. **Đây là thay đổi behavioural lớn** — trước thì mọi command sau PS crash đều fail "session not available".
- `pushFile` base64 validation (đóng vector PS-injection).
- EPIPE crash → graceful failure (stdin/exit handlers).
- `Math.min(a-b)` → `Math.abs(a-b)` cho Rect(point, point) selectors.
- `setWindow` dùng native handle foreground (trước là NaN no-op khi lookup theo name).
- 8 pattern handlers route qua `resolvePatternElement` để consistent W3C error surfacing.
- UWP path detection regex thắt chặt.
- `NotCondition` error message fix (đang copy-paste từ AndCondition).
- `$()` factory `&&` → `||`.

> **Cảnh báo review**: 1.1.9 vừa thêm 8 W3C commands vừa đổi behaviour PS lifecycle. Nếu nghi regression từ đây, đáng nghi nhất là **PS auto-restart** và **TransformPattern setWindowRect**.

## 1.1.11 — `6f060a6` "PowerShell session protocol overhaul"

**Đại tu PS protocol** — đây là chỗ user-visible behaviour thay đổi nhiều nhất.

### Session lifecycle
- Bỏ `-NoExit` khỏi spawn args. PS giờ exit khi stdin close (trước phải SIGKILL).
- `killProcessTree` khi forced shutdown → PS-spawned children không orphan.
- Thêm flag `powerShellTerminating`: in-flight command resolve silently khi teardown; queued commands fast-fail với `NoSuchDriverError` thay vì respawn PS cho từng item.
- Init / `setupRootElement` failure giờ kill spawned process + clear `this.powerShell` (trước để session half-initialised).
- Thêm child process `'error'` handler (trước crash Node trên spawn ENOENT).

### Command framing
- Per-command **UUID marker** trên cả stdout VÀ stderr (trước: 1 static char `0xF2EE`). Đóng marker collision và race stderr-after-stdout.
- User command bọc trong `& { ... }`. **Bug ở đây gây hang 1.1.11 → phải hotfix 1.1.12.**
- Marker detection check accumulated buffer (trước check chunk → miss khi marker split).

### Per-command buffer isolation
- Xoá `powerShellStdOut` / `powerShellStdErr` khỏi driver. Mỗi command sở hữu `CommandContext` riêng.
- Command queue check honour `powerShellTerminating` mid-queue.

### Return-code handling
- `$LASTEXITCODE` check sau user command → surface native-exe non-zero exit qua `[NativeExit] N` trên stderr (trước: silent success).
- **Capability mới `treatStderrAsError`** (default `true`). Khi `false`, plain stderr log debug và command resolve; `[NativeExit]` vẫn reject.
- `onClose` log không còn "Error code 0" trên clean exit.
- `sendIsolatedPowerShellCommand` hex display fix (trước render `0x-1` khi killed).

### Observability
- Init script split 9 named stages có timing logs.
- Per-command timing + first 80 chars debug.
- 17 unit tests mới trong `powershell-runtime.spec.ts`.

> **Đây là batch dễ nghi regression nhất.** Release note (`80bdb12`) đánh dấu 1.1.11 là **DEPRECATED — shipped broken**.

## 1.1.12 — `8edeb59` "fix session-start hang from `& { ... }` wrap"

**Critical hot-fix** cho 1.1.11.

**Nguyên nhân:**
1. `using namespace System.Windows.Automation` bên trong `& { ... }` **silent stall** stdin parser của PS → init script không bao giờ chạy hết.
2. Variables (`$cacheRequest`, `$elementTable`, `$rootElement`, …) sống trong subshell `& { }` → mất sau mỗi command.

**Fix:**
- Bỏ wrap `& { ... }`. User command gửi raw giữa `$LASTEXITCODE` reset và marker lines.
- Consolidate init về 1 combined script duy nhất (1 lần `sendPowerShellCommand`) → `using namespace` apply cho toàn bộ + variables persist.
- Bỏ per-init-stage timing logs (không đáng vì gây breakage).

**Trade-off**: user command malformed (open pipe, unclosed brace) giờ absorb marker → 60s timeout thay vì silent corruption. Là caller bug, làm visible.

## 1.1.14 — `a9da981` (`686027a` + `d865870` + `8bb2f1b`) "PowerShell-layer fixes"

Trigger từ production QAOFF-309 TC_009: long-running UIA find trên SecureAge dialog → 60s timeout → 3× `ConnectTimeoutError` ở teardown.

### `686027a` — fix chính
- **Non-blocking `killProcessTree`**: `spawn + unref` thay vì `execSync('taskkill')` → event loop không bị freeze. **Direct fix cho ConnectTimeoutError cascade.**
- `setupRootElement`: OR-guard ngắn mạch khi chỉ có `appTopLevelWindow` (no app) → silent set `$rootElement=null`. Fix bằng split branches null / none / root / app / handle.
- `Set-Location`: escape single quote trong `appWorkingDir`.
- Timeout fallback: `??` + validation số dương (trước `||` swap 0).
- Teardown: in-flight commands reject `NoSuchDriverError` (trước resolve `''` → misleading NoSuchElementError).
- Marker scan: track `stdoutScanIdx` / `stderrScanIdx` → tail-scan O(n) (trước O(n²)).
- **Default `powerShellCommandTimeout`: 60_000 → 300_000 ms.**
- Deploy script `build_deploy_restart.ps1`: stop-Appium block bọc try/catch `exit 0` (idempotent).

### `d865870` — 4 latent bugs
- `user32.sendMouseMoveInput`: `cubic-bezier(...)` capture group dùng `.groups?.[0]` trên unnamed → luôn undefined → fallback linear. Switch sang `match[1]`.
- `PSCultureInfo` type guard: `typeof !== 'string' || (typeof === 'number' && < 0)` → numeric overload unreachable + emit PS thiếu dấu phẩy `::new('en-US'$true)`.
- `PropertyCondition` OrientationType / AutomationHeadingLevel: integer-bounds check đọc length enum kia (swap nhầm).
- `converter.ts` Rect/Point parsing (C1, C3, C4, C5): callback `replaceAll` khai báo thiếu args + dùng `.groups` trên unnamed → NaN. Rewrite consume flat captures.

### `8bb2f1b` — code-quality
- `executeKeys`: validate set-ness bằng `!== undefined` thay `!!` → `pause: 0`, `text: ''`, `virtualKeyCode: 0` không còn bị reject.
- `changeRootElement`: escape single quote trong `normalizedPath` + `appArguments` trước khi nhúng PS single-quote.
- `pullFolder`: bọc `Compress-Archive` / `ReadAllBytes` trong try/finally → cleanup staging zip.
- `DeferredStringTemplate.format`: rewrite loop để không emit trailing undefined.

## 1.1.17 — `2e9e767` (`bcea92e`) "stability fixes for long-run and stress"

### `lib/commands/functions.ts` (A18)
- `Get-PageSource` **rewrite thành iterative BFS** (queue `(element, parentXmlElement)`). Stack depth O(1) thay vì recursive (PS function-call stack ~5000 frame ceiling crash trên deep desktop tree). Public signature unchanged.
- Find-AllDescendants trên desktop root giờ stream qua TreeWalker thay vì `FindAll(Subtree)` → loại `System.OutOfMemoryException` từng kill SecureAge test session (và parent Appium server).

### `lib/powershell/elements.ts` + `lib/commands/powershell.ts` (A22)
- `$elementTable` giờ **FIFO-bounded 10_000 entries** qua `$elementTableOrder` Queue. Trước grow vô hạn → PS OOM. `ensureElementResolved` re-find evicted runtime ids theo RuntimeId.

### `lib/commands/powershell.ts` + `lib/driver.ts`
- **Command queue depth cap = 200.** Thêm field `powerShellQueueDepth` trên driver; `sendPowerShellCommand` reject khi cap → Promise chain không grow vô hạn. Decrement trong try/finally.

### `lib/driver.ts`
- `deleteSession` explicit reset `commandQueue` (Promise.resolve()), `powerShellQueueDepth` (0), `powerShellCommandContext` (undefined), keyboard state flags.

E2E mới `session-stress.e2e.spec.ts`: 30× create / use / delete, assert không leak (last-5 avg ≤ 3× first-5), queue-cap reject path.

## 1.1.19 — `9947276` "screenshot OOM cascade fix + log readability"

### `lib/commands/functions.ts`
- Screenshot **catch path không touch GDI+ nữa**: return pre-encoded 1×1 PNG literal thay vì allocate `Drawing.Bitmap` + `Graphics` + `MemoryStream`. Loại secondary OOM trong catch block (làm pwsh crash `0xE0434352`).
- Validate bitmap dims trước allocation: reject zero / negative / >32767 (tránh GDI+ surface oversized / Rect.Empty roots như OOM).

### `lib/commands/powershell.ts`
- Khi stderr match `OutOfMemoryException` / `StackOverflowException` / `ExecutionEngineException` / `AccessViolationException` → kill PS child để command kế tiếp respawn clean session. Đóng cascade 2–3 command quan sát ở production (`df9b46fe` session).
- Log full decoded command (`decodePwsh`) thay vì 80-char base64 slice; bỏ duplicate `[PowerShell Raw Command]` lines.
